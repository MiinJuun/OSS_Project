## 1. Business Purpose

최근 외식 및 관광 시장에서 소비자들은 방문 전 온라인 플랫폼의 리뷰를 확인하는 것을 필수로 여깁니다. 이는 소비자의 의사결정에 절대적인 영향을 미치고 있습니다. 그러나 대표적인 플랫폼인 네이버 지도의 경우, 영수증 리뷰 이벤트와 같은 보상형 마케팅과 특정 업체에 대량 리뷰를 청탁하는 마케팅 업체의 개입으로 인해 무분별한 긍정 평가가 양산되는 현상이 고질적인 문제로 자리 잡았습니다. 특히 최근 네이버가 폐지했던 5점 만점의 별점 제도를 다시 부활(2026년 4월 예정)시키기로 결정함에 따라, 과거 심각했던 ‘별점 인플레이션’과 어뷰징(조작) 문제가 다시금 수면 위로 떠오를 것이 자명해졌습니다.

소비자들은 조작된 평점을 피하기 위해 비교적 솔직하고 날 것의 리뷰가 남겨진 카카오맵 등 타 플랫폼을 통해 교차 검증(Cross-check)을 수행합니다. 이는 불필요한 정보 탐색 비용과 피로도를 유발합니다. 더불어 최근에는 생성형 AI를 활용해 사람이 쓴 것처럼 자연스러운 가짜 리뷰까지 대량으로 쏟아지면서, 소비자가 진짜와 가짜를 걸러내기란 불가능에 가까워졌습니다.

본 서비스 RE:VIEW는 이러한 정보의 비대칭성과 플랫폼 정책 변화에 따른 혼란을 기술적으로 해결하기 위해 기획되었습니다. 다중 플랫폼의 리뷰 데이터를 수집 및 통합하여 다음과 같은 본질적인 분석을 수행합니다.

* **별점 존재 시:** 통계적 이상 탐지로 플랫폼 간 ‘평점 괴리율’ 계산 및 조작 패턴 시각화
* **텍스트/작성자 위주 시:** NLP 기반 감성 분석 및 작성자 활동 패턴 분석을 통한 어뷰징 계정 선제 필터링

본 프로젝트의 궁극적인 목적은 플랫폼 정책 변화에 관계없이 데이터의 본질을 분석하여 사용자에게 신뢰할 수 있는 **‘진짜 여론’ 대시보드**를 제공하는 것입니다. 추가로 본 서비스의 **‘주요 타겟 시장’**(Target Market)은 맛집, 카페, 관광지 등을 방문하기 전 블로그나 영수증 리뷰의 진위 여부를 의심하고 피로감을 느끼는 20~30대 외식/관광 소비자층입니다

| **6** | **어뷰징 알림 확인** | User | 단기간 리뷰 급증이나 조작(어뷰징)이 의심될 때 시스템이 화면에 띄우는 경고 알림을 확인한다. |

| **8** | **분석 리포트 공유** | User | 대시보드 화면 전체를 PDF 파일로 변환하여 저장하거나 공유한다. |
| **9** | **타겟 장소 비교 분석** | User | 두 장소의 베이지안 평균 및 세부 감성 지표를 나란히 대조하여 우위를 평가한다. |

## 5. Problem Statement

### 5.1. Problems to be Considered (Including Technical Difficulties)

**[비즈니스 및 사회적 문제]**
현재 온라인 장소 리뷰 플랫폼은 보상형 마케팅(리뷰 이벤트 등)으로 인한 무분별한 긍정 평가 양산이 이루어지고 있습니다. 더불어, 생성형 AI를 통해 실제 사람이 쓴 것처럼 자연스러운 문맥으로 생성되는 ‘가짜 리뷰’가 범람하면서 리뷰 데이터의 신뢰도가 심각하게 상실되고 있습니다. 특히 2026년 4월 네이버의 별점 제도가 부활하면서 과거의 극심한 ‘평점 인플레이션’ 현상까지 재현될 조짐을 보이고 있습니다. 
이로 인해 소비자들은 단일 플랫폼의 평가를 믿지 못하고 타 플랫폼을 직접 오가며 교차 검증을 하는 불필요한 시간적 비용을 지출하게 됩니다. 결과적으로 고도화된 어뷰징과 정보의 비대칭성으로 인해 소비자의 합리적인 의사결정이 방해받고 있는 것이 핵심 문제입니다.

**[시스템 구현 시 고려해야 할 기술적 난제]**
위의 문제를 해결하기 위한 대시보드를 구현하는 과정에서 다음과 같은 기술적 어려움을 고려하고 해결해야 합니다.

| Technical Difficulty | Description |
| :--- | :--- |
| **동적 웹 페이지 크롤링의<br>불안정성 및 차단 위험** | 네이버 지도와 카카오맵은 SPA(Single Page Application)로 구성되어 있어 정적 HTML 파싱만으로는 데이터를 얻을 수 없습니다. 동적 렌더링 대기 시간, DOM 구조의 잦은 변경, 그리고 봇(Bot) 탐지로 인한 트래픽 차단(IP Block) 위험을 우회하는 안정적인 수집 파이프라인 구축이 필요합니다. |
| **비정형 텍스트 감성 분석의<br>문맥적 한계 (NLP)** | "맛없을 수가 없네요", "다시는 안 갈 듯ㅋㅋ" 등 한국어 특유의 반어법, 은어, 문맥적 뉘앙스를 단순 키워드 매칭으로는 정확히 분류하기 어렵습니다. 텍스트의 숨은 의도를 파악하고 긍/부정 확률로 수치화하는 정교한 모델링이 필요합니다. **이를 해결하기 위해 한국어 자연어 처리에 우수한 성능을 보이는 KoBERT 모델을 도입하고, Python API를 통해 핵심 시스템과 연동하여 감성 분석의 정확도를 극대화할 계획입니다.** |
| **다중 프로세스 실시간<br>통합 처리 성능 확보** | 사용자의 검색 요청 즉시 '양대 플랫폼 동시 크롤링 $\rightarrow$ NLP 감성 분석 $\rightarrow$ 통계적 이상 탐지 $\rightarrow$ UI 렌더링'이라는 무거운 작업들이 백그라운드에서 직렬로 실행될 경우 치명적인 병목 현상이 발생하므로, 이를 최적화할 비동기 처리 전략이 필수적입니다. |
| **표본 수 불균형에 따른<br>통계적 오류 (희소 데이터)** | 리뷰가 3개뿐인 업체의 만점(5.0)과 리뷰가 1,000개인 업체의 4.5점을 단순 비교하면 통계적 왜곡이 발생합니다. 데이터가 적을 때 극단적인 값으로 튀는 현상을 방지하기 위해 베이지안 평균(Bayesian Average)과 같은 보정 알고리즘 적용이 필요합니다. |

## 6. Glossary

* **교차 검증 (Cross-check):** 여러 자료를 토대로 정확성을 높이는 기본 검사법. (네이버 vs 카카오 데이터 대조)
* **괴리율 (Discrepancy Rate):** 시장 가격과 기준 가격의 차이 비율. (보상형 이벤트 등에 따른 플랫폼 간 감성 수치 차이)
* **자연어 처리 (NLP):** 인간의 언어를 컴퓨터가 이해하고 처리하도록 연구하는 학문.
* **감성 분석 (Sentiment Analysis):** 머신러닝을 사용하여 텍스트의 주관적인 감정적 어조를 확률 수치로 판별하는 기술.
* **어뷰징 (Abusing):** 부당한 방법으로 이득을 챙기는 행위. (영수증 리뷰 조작, AI 생성 가짜 리뷰 등)
* **베이지안 평균 (Bayesian Average):** 데이터가 적을 때 전체 평균(사전 정보)을 반영해 추정치를 보정하는 방법. (리뷰 적은 업체의 극단적 평점 방지)
* **리뷰 최신성 가중치:** 과거의 평가가 현재를 왜곡하는 것을 막기 위해 최근 리뷰에 더 높은 가중치를 부여하는 기법.
    > **Ex)** 2년 전엔 맛있었으나 최근 주인이 바뀌어 혹평이 쏟아지는 경우, 전체 평균의 왜곡을 방지함.
* **대시보드 (Dashboard):** 여러 데이터와 분석 결과를 한 화면에서 직관적으로 파악할 수 있도록 구성된 사용자 인터페이스.
* **메타데이터 (Metadata):** 리뷰 텍스트 자체가 아닌, 리뷰에 부여된 별점, 작성 시간, 작성자 정보 등 원본 데이터를 설명하는 구조화된 부가 데이터.
* **크롤링 (Crawling):** 무수히 많은 컴퓨터에 분산 저장되어 있는 문서를 수집하여 검색 대상의 색인으로 포함시키는 기술. 본 프로젝트에서는 웹 페이지의 리뷰 데이터를 추출하는 행위를 뜻함.
* **SPA (Single Page Application):** 단일 웹 페이지로 구성되어, 사용자와 상호작용할 때마다 페이지 전체를 새로고침하지 않고 필요한 데이터만 동적으로 렌더링하는 웹 애플리케이션 구조. (네이버 지도, 카카오맵의 웹 구조)
* **KoBERT:** SK텔레콤에서 대규모 한국어 데이터를 학습시켜 개발한 한국어 특화 자연어 처리(NLP) 인공지능 모델. 한국어의 복잡한 뉘앙스와 문맥을 파악하는 데 탁월함.
* **Python API:** 다른 프로그램에서 파이썬(Python)으로 작성된 기능(머신러닝 모델 등)을 호출하고 사용할 수 있도록 만들어진 연결 인터페이스.
