## 1. Introduction
본 Analysis 문서는 앞선 Conceptualization 단계에서 정의된 'RE:VIEW' 대시보드의 비즈니스 목적을 실제 소프트웨어 시스템의 요구사항으로 
구체화하기 위해 작성되었습니다. 최근 온라인 플랫폼의 보상형 마케팅과 AI 생성 가짜 리뷰로 인해 소비자의 정보 비대칭성 및 교차 검증 피로도가 
극심해지고 있습니다. 본 분석 단계에서는 이러한 고질적인 문제를 기술적으로 해결하기 위해 시스템의 논리적으로 '무엇(What)'을 수행해야 하는지 
명세합니다. 
'RE:VIEW'의 핵심 목표인 통계적 이상 탐지와 NLP 기반 감성 분석을 통한 '진짜 여론' 도출을 실현하기 위해, 본 문서는 SRUP(Simplified Rational 
Unified Process) 방법론에 입각하여 시스템을 체계적으로 모델링합니다.

본 RE:VIEW 프로젝트가 가지는 주요 특징과 기대 효과는 다음과 같습니다.

* 프로젝트의 의의 (Significance): 단순한 리뷰 정보의 취합을 넘어, 한국어 문맥 분석에 특화된 KoBERT 모델과 표본 불균형을 보정하는 베이지안 평균
등 고도화된 기술을 융합한다는 데 있습니다. 이를 통해 왜곡된 평점 시스템 속에서 '진위가 검증된 객관적 지표'를 도출해 내어 정보의 비대칭성을 해소한다는
큰 사회적, 기술적 의의를 지닙니다.

* 프로젝트의 유용성 (Usefulness): 사용자가 타 플랫폼을 직접 오가며 일일이 대조해야 했던 수고를 단 한 번의 장소 검색으로 해결합니다. 양대 플랫폼
(네이버 지도, 카카오맵)의 데이터를 통합 수집하여 '감성 괴리율'과 공통으로 언급된 '교차 검증 핵심 키워드'를 직관적인 대시보드 형태로 제공함으로써,
사용자의 불필요한 정보 탐색 시간을 획기적으로 단축하고 합리적인 의사결정을 돕습니다.

* 프로젝트의 확장성 (Expandability): 데이터 수집(웹 크롤링), 비정형 텍스트 감성 분석(NLP), 통계적 이상 탐지 및 대시보드 시각화 모듈이 각각 독립적인 
파이프라인으로 설계되어 높은 모듈 독립성을 확보합니다. 이러한 아키텍쳐는 향후 외식 및 관광 도메인뿐만 아니라 배달 애플리케이션, 숙박 플랫폼 등 리뷰 데이터
가 존재하는 타 분야로도 시스템을 유연하게 연동하고 확장할 수 있는 강력한 기반이 됩니다.

이를 위해 본 문서에서는 시스템과 사용자 간의 상호작용을 정의한 Use Case Analysis, 문제 영역 내의 핵심 객체를 식별한 Domain Analysis, 그리고 최종
사용자의 시작적 경험을 구상한 User Interface Protoype을 차례로 기술합니다.

## 4. User Interface Prototype

본 단계에서는 RE:VIEW 시스템의 최종 사용자가 경험하게 될 시각적 인터페이스를 정의합니다. 시스템의 주요 기능(Use Case)들이 화면 상에서 어떻게 구현되고 상호작용하는지 예비 사용자 매뉴얼(Preliminary User Manual)의 형태로 기술합니다.

### 4.1. 메인 검색 화면 (Main Search View)
<img width="2556" height="1336" alt="image" src="https://github.com/user-attachments/assets/431fd2ff-0ac6-4f4e-b6fe-162ccecdd144" />

**[화면 설명 및 사용 매뉴얼]**
사용자가 대시보드 시스템에 접속했을 때 가장 먼저 마주하는 초기 화면입니다. 복잡한 정보 없이 직관적인 검색을 유도하도록 설계되었습니다.
* **통합 검색창:** 화면 중앙에 위치합니다. 분석하고자 하는 타겟 장소(식당, 카페 등)의 이름을 입력하면, 하단의 네이버 지도 및 카카오맵 API와 연동된 자동완성 후보군 리스트가 드롭다운 형태로 나타납니다. (관련: UC-01 장소 검색)
* **검색 버튼 (돋보기 아이콘):** 후보군 중 하나를 마우스로 클릭하거나 엔터키를 누르면, 시스템이 즉시 백그라운드 크롤링 및 분석을 시작하며 대시보드 화면으로 전환됩니다.

### 4.2. 통합 분석 대시보드 화면 (Integrated Analysis Dashboard)
<img width="1277" height="671" alt="image" src="https://github.com/user-attachments/assets/626af7ca-051a-472d-9992-641f76f32365" />

**[화면 설명 및 사용 매뉴얼]**
RE:VIEW 시스템의 핵심 가치를 제공하는 메인 결과 화면입니다. 수집된 다중 플랫폼의 리뷰 데이터와 AI 감성 분석 결과가 한눈에 들어오도록 배치됩니다.
* **어뷰징 경고 배너 (Alert Area):** 화면 최상단에 위치합니다. 단기간 리뷰 급증 등 조작이 강하게 의심될 경우, 눈에 띄는 붉은색 배너로 "보상형 마케팅(리뷰 조작)이 강하게 의심되는 장소입니다"라는 경고를 표출합니다. 평소(정상)에는 숨김 처리됩니다. (관련: UC-06 어뷰징 알림 확인)
* **상단 필터 및 내보내기 툴바:** * **조건 필터:** 1개월, 6개월, 전체 등 리뷰 수집 기간을 설정할 수 있는 드롭다운 메뉴가 좌측 상단에 제공됩니다. (관련: UC-04 분석 조건 설정)
  * **PDF 공유 버튼:** 우측 상단의 아이콘을 클릭하면 현재 렌더링된 화면 전체를 PDF 리포트로 다운로드할 수 있습니다. (관련: UC-08 분석 리포트 공유)
* **감성 괴리율 차트 (Center View):** 네이버 지도와 카카오맵 각각의 NLP 감성(긍/부정) 수치를 도넛 차트로 보여주고, 두 플랫폼 간의 여론 온도 차이(괴리율 %)를 중앙에 큰 텍스트로 강조합니다. (관련: UC-05 감성 괴리율 조회)
* **교차 검증 하이라이트 (Bottom View):** 두 플랫폼에서 공통으로 가장 많이 언급된 진짜 장점과 단점 키워드(예: "주차장 넓음", "위생 불량")만을 필터링하여 워드 클라우드 또는 태그 리스트 형태로 제공합니다. (관련: UC-07 교차 검증 키워드 조회)
* **통합 원본 리뷰 패널 (Side View):** 차트 양옆 또는 하단에 각 플랫폼에서 수집된 실제 리뷰 텍스트가 스크롤 가능한 리스트로 제공됩니다. (관련: UC-02 리뷰 데이터 통합 조회)
  * **신뢰도 배지:** 각 리뷰 작성자 아이디 옆에 초록색(정상), 노란색(주의), 빨간색(어뷰징 의심)의 작은 방패 모양 배지가 부착되어 리뷰의 신뢰도를 직관적으로 판별할 수 있습니다. (관련: UC-03 작성자 신뢰도 분석)
